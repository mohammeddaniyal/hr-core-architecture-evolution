HR Core: Architecture Evolution

A project demonstrating the evolution of a Java web architecture — from raw Servlet-based rendering to a fully decoupled Spring Boot SPA.

Each stage represents a deliberate architectural refactor to solve real limitations discovered in the previous implementation.

Author: Mohammed Daniyal

Raw Servlet Monolith

Stage 1
Java Servlets DAO Pattern Custom Java CLI

The baseline architecture. Database interactions are isolated in a DAO layer, but UI logic and business rules are tightly coupled inside Servlets. To reduce the friction of manually generating HTML inside Java, a custom CLI tool (PWPrintln) was created to convert HTML markup into PrintWriter.println() statements.

MVC Architecture

Stage 2
JSP Reflection API Custom Tag Engine

The architecture shifts to a strict Model-2 MVC pattern using JSP for the view layer. A custom Tag Library (TLD) was implemented to eliminate Java scriptlets, enabling declarative UI binding via Java Reflection and enforcing Synchronizer Token security at the view level.

Stateless Client-Server Architecture

Stage 3
Vanilla JS AJAX JSON API DOM Templating

The backend stops rendering HTML and exclusively provides a JSON-based API. The frontend relies on raw XMLHttpRequest and a custom Vanilla JS cloneNode templating engine, eliminating synchronous page reloads and decoupling presentation from data.

Spring Boot SPA Architecture

Stage 4
Spring Data JPA Vanilla JS Router Spring Security

The backend is rebuilt using Spring Boot and Spring Data JPA, replacing manual JDBC with Hibernate ORM. The frontend evolves into a Single Page Application using a custom History API router. Authentication and Spring Security RBAC dynamically control UI behavior based on user roles.
